## 〈 Featured Projects 〉

<details>
<summary><b>🏦 Civic Vault — Banking Management System</b></summary>

<br/>

A production-grade full stack banking web application with secure authentication, atomic fund transfers, transaction history, and a role-based admin panel.

| Attribute | Details |
|-----------|---------|
| **Stack** | Python, Django REST Framework, SimpleJWT, React JS, Axios, Bootstrap 5, SQLite |
| **Security** | JWT authentication, role-based access control, token expiry handling |
| **Data Integrity** | `transaction.atomic()` ensures zero data loss during fund transfers |
| **Admin Controls** | Freeze/unfreeze accounts, view all users and transactions |
| **Architecture** | Decoupled REST API backend + React SPA frontend |
| **Repository** | [github.com/Necro-noirrr/Civic-Vault](https://github.com/Necro-noirrr/Civic-Vault) |

**Key engineering decisions:**
- Used Django's `transaction.atomic()` to wrap all fund transfer operations — ensuring debit and credit either both succeed or both rollback, preventing any state of partial transfer
- Implemented `IsAuthenticated` and `IsAdminUser` permission classes at the view level for granular access control
- Built a custom `RegisterSerializer` that auto-creates a linked `BankAccount` on every user registration using Django signals pattern
- Configured Axios interceptors to auto-attach JWT tokens to all outgoing API requests — single point of auth management for the entire frontend

</details>

## 〈 Engineering Principles 〉

| Principle | Practice |
|-----------|----------|
| **Data Safety** | Atomic transactions for all state-changing operations |
| **Security First** | JWT auth, role-based permissions, no plain-text secrets |
| **API Design** | RESTful endpoints, proper HTTP status codes, serializer validation |
| **Clean Code** | Separation of concerns — models, serializers, views, URLs decoupled |
| **Version Control** | Meaningful commits, structured repositories, documented codebases |
